1. コアコンセプト
「意思を持つAIの挙動を、箱庭でニヤニヤ観察するパズルシミュレーション」
プレイヤーは直接手を下さず、小人視点で島を駆け巡りながら、上から落とす「エサ」の誘導だけで、過酷な環境（細い橋、風）の先にあるアイテムをローポリの生き物たちに獲得させる。

2. 実装済み機能（DONE）
[x] フィールド・環境配置: Blenderからエクスポートした島モデル（Island.glb）の配置と、メッシュに完全に追従する当たり判定（StaticBody3D）の一発生成。

[x] 小人フライトカメラ（Minecraft風）: - WASDによる移動、Space/Shiftによる垂直移動の完全分離。

飛行モード（空中）：心地よい滑らかな慣性移動。

歩行モード（地上）：慣性を完全に排除したキビキビ移動、重力による落下、ジャンプ。

スペースキー2回押し（ダブルタップ）での飛行/歩行モードのシームレスな切り替え。

プレイヤーのコリジョン（CharacterBody3D）の実装による、地面やすり抜けの完全防止。

FOV（視野角）の調整による小人スケール感の演出。

3. 現在の開発バックログ（NEXT）
🍏 フェーズ1：エサやりシステム（予測線付き）
[ ] エサ（Food.tscn）の作成: 物理演算（RigidBody3D）で転がる小さな美味しそうな実。

[ ] 着弾予測線の描画: カメラの視線の先（画面中央）から地面に向かって、白い点線のレーザー（RayCast3D）を照射する。点線は下に向かってスーッと動くアニメーション付き。

[ ] 投下ロジック: クリックすると、予測線の先端（地面の交点）の少し上空にエサをスポンさせて落とす。

📊 フェーズ2：生物観察 UI ＆ ズームイン機能
[ ] 生物一覧UI（CanvasLayer）: - 画面隅に、現在生存している個体、死亡した個体の一覧をリスト表示。

リストには、各個体の状態（生存/死亡）や、遺伝した特性タグ（[目がいい]、[鼻がきく]など）を表示。

[ ] カメラズームイン機能:

UI内の生物リストをクリックするか、3D上の生き物を直接クリックすると、カメラがその生き物の元へ滑らかに移動し、追跡（ストーキング）モードに入る。

🛒 フェーズ3：ショップ ＆ メタゲーム要素
[ ] Resource（リソース）による強化データ設計: Trait.cs（特性タグ）や ShopItem.cs をUnityのScriptableObjectと同じようにデータ資産化。

[ ] ショップUI: アイテム獲得や間引きで得たポイントを使い、「一度の生成数」「遺伝の引き継ぎ確率」「時の加速スライダー上限」「ナイフ」等を購入・解放できる仕組み。

4. 将来の拡張：強化学習（AIの脳）
